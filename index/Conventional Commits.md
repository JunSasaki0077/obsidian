
## Conventional Commitsとは

[[GitHub]]上に[[commit]]する際に
人間と機械が読みやすく、意味のあるコミットメッセージにする
をコンセプトにしているcommitメッセージの仕様

commitメッセージの前に型を指定することにより
パット見てわかりやすくしている。
例：
```md
feat: ログイン機能を実装
```

## 型一覧

1. fix:  バグにパッチを当てた際に使用
2. feat: 新しい機能を追加した際に使用
3. docs: ドキュメントのみを変更した際に使用
4. perf: パフォーマンスを向上させるコード変更
5. build:型 
6. test: 不足しているテストを追加、既存のテストの修正
   
7. BREAKING CHANGE:型
   APIの破壊的変更を行った際に使用
   または型の後ろに`!`を使用することで危ないよと明示的に示す
