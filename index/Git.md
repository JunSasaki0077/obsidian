#Git #IT 

## Gitとは


一言でいうと`ファイルのバージョン管理が簡単にできるツール`です。

**誰が、どのファイルに、何を書き込んだのか、何を削除したのか、
編集前とどう変わったのか**を履歴として残しておき、
必要であればそのバージョンまで戻ることもできる。

様々なコマンドがあり、そのコマンドを用いて
バージョン管理を行う

## よく使うコマンド

#### 確認系
- git status
	- 今の作業状態を確認する
	- 変更したファイルが / [[add]]済か未[[add]]かがわかる
- git log
	- [[commit]]履歴を見ることができる
	- 誰が・いつ・何をしたか確認
- git diff
	- 変更内容の差分を見る
	- [[commit]]前のセルフレビューに必須

#### [[ブランチ]]操作
- git branch
	- ブランチ一覧を見る
	- 今どのブランチにいるか確認
- git branch ブランチ名
	- 新しいブランチを作る
- git switch ブランチ名
	- ブランチを切り替える
- git switch -c ブランチ名
	- 新しいブランチの作成 + 切り替えを同時に行う

#### [[リモート]]連携系
- git fetch
	- [[リモート]]の最新情報を取得
	- 安全確認ように使用
- git pull
	- 最新情報を取得 + 反映を一気に行う
	- 便利だけど事故りやすいので使わない
- git push
	- ローカルの[[commit]]を[[リモート]]へ送信

#### やり直し・修正系（事故対応）
- git restore ファイル名
	- ファイルを変更前に戻す
	- `commit前限定！`
- git restore --staged ファイル名
	- [[add]]を取り消す
- git reset HEAD~1
	- 直前の[[commit]]を残す(履歴は残る)
- git revert commitID
	- commitを`打ち消すcommit`を作る
	- **実務ではreset HEAD~1よりこっちが安全**

#### マージ・履歴整理
- git merge ブランチ名
	- [[ブランチ]]を合体させる
	- [[プルリクエスト]]後によく使う
- git rebase
	- 履歴をきれいに並べ直す
	- `使い方ミスると地獄`
- rebaseはチームルール厳守！
- わからないうちは触らない

#### 一時避難・便利技
- git stash
	- 作業中の変更を一時保存
	- `今すぐやらないといけない別の作業頼まれたときの神コマンド`
- git stash pop
	- 避難した作業を戻す

#### トラブルシュート系
- git cherry-pick commitID
	- 特定の[[commit]]だけ持ってくる
	- バグ修正の横取りに使う
- git blame ファイル名
	- 誰がその行を書いたか確認する
	- **犯人探し用(優しく使おう)**
